# 视频转文字工具 (Video Transcriber)

## 1. 项目概述

**项目名称**: Video Transcriber  
**项目类型**: 本地桌面应用 (PyQt5)  
**核心功能**: 视频文件 → 音频提取 → Whisper CUDA 转录 → Word 文档（带标点、分段）  
**目标用户**: 需要将视频内容转换为文字文档的用户

## 2. 核心功能需求

### 2.1 视频处理流程

| 阶段 | 功能 | 工具 |
|------|------|------|
| 1 | 选择视频文件 | PyQt5 文件对话框 |
| 2 | 音频提取 | ffmpeg |
| 3 | 语音转文字 | faster-whisper small (CUDA) |
| 4 | 添加标点符号 | 内置规则 + punkt tokenizer |
| 5 | 智能分段 | 基于语义和时长 |
| 6 | 导出 Word | python-docx |

### 2.2 进度反馈机制

- **阶段进度条**: 显示当前阶段（音频提取/转录/格式化）
- **片段进度**: 显示当前片段和总片段数
- **预估时间**: 基于处理速度估算剩余时间
- **实时日志**: 显示详细处理状态

### 2.3 容错与恢复机制

| 机制 | 描述 |
|------|------|
| 断点续传 | 保存处理进度到 JSON，重启后可继续 |
| 防止误杀 | 处理中禁用关闭按钮，需确认 |
| 卡顿检测 | 30秒无进度更新视为卡住 |
| 手动重启 | 卡住时可手动选择重启当前任务 |
| 临时文件保留 | 中断后可从断点恢复 |

### 2.4 内存优化

- **分段处理**: 音频按 5 分钟分段处理
- **及时写入**: 每完成一段立即写入 Word
- **GPU 内存管理**: 设置合理的 batch_size
- **内存监控**: 实时显示内存使用情况

## 4. 技术架构

### 4.1 模块划分

```
video_transcriber/
├── video_transcriber.py    # 主程序入口
├── core/
│   ├── audio_extractor.py  # ffmpeg 音频提取
│   ├── transcriber.py      # Whisper 转录核心
│   ├── punctuation.py      # 标点添加模块
│   └── docx_exporter.py    # Word 导出
├── utils/
│   ├── progress_tracker.py # 进度追踪器
│   ├── checkpoint.py       # 断点续传
│   └── memory_monitor.py   # 内存监控
└── ui/
    └── main_window.py     # 主窗口
```

### 4.2 数据流

```
视频文件 → FFmpeg 提取音频 → 分段处理 
                              ↓
                         Faster-Whisper 转录
                              ↓
                         标点添加 + 分段
                              ↓
                         增量写入 Word
                              ↓
                         保存进度 + 完成
```

### 4.3 断点续传数据结构

```json
{
  "video_path": "/path/to/video.mp4",
  "output_path": "/path/to/output.docx",
  "config": {
    "model": "small",
    "chunk_duration": 300,
    "add_punctuation": true
  },
  "state": {
    "current_chunk": 5,
    "total_chunks": 12,
    "transcribed_text": ["chunk 1 text", "chunk 2 text", ...],
    "last_update": "2024-04-15T10:30:00"
  }
}
```

## 5. 验收标准

### 5.1 功能验收

| 编号 | 功能点 | 验收条件 |
|------|--------|----------|
| F1 | 文件选择 | 支持 mp4/avi/mkv/mov 等常见视频格式 |
| F2 | 音频提取 | ffmpeg 成功提取 wav/mp3 |
| F3 | 语音转写 | faster-whisper 正确转写中文语音 |
| F4 | 标点添加 | 自动为转写文本添加句号、逗号等 |
| F5 | 智能分段 | 根据语义完整性进行段落划分 |
| F6 | Word 导出 | 生成可读的 .docx 文件 |
| F7 | 进度显示 | 实时更新进度条和预估时间 |
| F8 | 断点续传 | 重启后从上次位置继续 |
| F9 | 防止误杀 | 处理中关闭需二次确认 |
| F10 | 错误处理 | 异常情况有友好提示 |

### 5.2 性能要求

- 音频提取速度: 与视频时长相当
- 转录速度: >= 10x 实时 (GPU)
- 内存占用: 峰值不超过 8GB
- 启动时间: < 5 秒

## 6. 参考设计原则

本项目参考 Anthropic 的《Harness Design for Long-Running Application Development》:

1. **冲刺合约**: 每个处理阶段有明确的完成标准
2. **迭代式反馈**: 实时进度反馈让用户了解任务状态
3. **容错阈值**: 异常情况自动暂停并提示用户
4. **状态持久化**: 结构化保存任务状态，支持断点续传
5. **上下文重置**: 支持从检查点恢复，避免重复工作
